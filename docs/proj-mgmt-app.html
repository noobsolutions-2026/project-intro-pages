Project Management Application

Node.js web application with Microsoft Entra ID (Azure AD) authentication using OAuth 2.0 and OpenID Connect.

Project Management Application provides secure authentication via Microsoft Entra ID with session-based user management. Secrets are stored in AWS SSM Parameter Store and the app can be deployed containerized to AWS ECS/Fargate or run locally for development.

  • Microsoft Entra ID (Azure AD) authentication with Passport.js
  • Secrets in AWS SSM Parameter Store
  • Containerized with Docker and deployable to ECS/Fargate
  • Role-based access control backed by DynamoDB
  • S3-backed file storage for project files

Key User Cases

1. Create Project

Users can initiate new projects within the application. This involves defining project details such as name, description, timelines, and initial team members. Upon creation, a project serves as the central hub for all related tasks, assessments, and progress tracking.

2. Assign Assessments

Once a project is established, assessments can be assigned to individual team members or roles within that project. These assessments can be tasks, milestones, or specific deliverables. The system ensures that each assessment is linked to a project and assigned to a responsible party, facilitating clear ownership and progress monitoring.

Relationship between Project Creation and Assessment Assignment

The core workflow within the Project Management Application begins with the creation of a project. Only after a project is successfully set up can assessments be defined and assigned. This sequential relationship ensures that all work (assessments) is contextually tied to a larger objective (the project).
