Assessment Status Workflow

Every assessment progresses through a structured four-stage lifecycle

01
Ongoing
Team is actively conducting the security assessment. Work is in progress.
02
Finding Report Released
Initial findings delivered to client for review and action.
03
Revisit Completed
Remediation re-testing performed; verified findings updated accordingly.
04
Assessment Completed
All work done. Project auto-completes when every assessment reaches this stage.
✓  When all assessments complete → project auto-updates to 'Completed'. Zero manual steps required.

Enterprise Security & Integrations

M
Microsoft Entra ID SSO
OAuth 2.0/OIDC via Microsoft Entra ID. Zero friction for corporate Microsoft 365 environments.
C
CSRF Protection
Every non-auth endpoint protected with CSRF tokens via csurf middleware on all routes.
R
Rate Limiting
5 auth attempts / 15 min; 100 API calls / 15 min in production to prevent automated abuse.
A
AWS Cloud Infrastructure
DynamoDB, S3, SSM Parameter Store. Secrets loaded at runtime — never hardcoded anywhere.
P
Parameterised DynamoDB
All database operations use expression attribute names/values — injection risk fully eliminated.
S
S3 Presigned URLs
Time-limited secure download links; no public buckets. Paths enforce project-UID scoping at all times.
