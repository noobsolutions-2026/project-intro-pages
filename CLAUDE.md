# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A static GitHub Pages site that hosts project introduction pages. Pages are hand-authored (or LLM-generated) self-contained HTML files under `docs/`, organized by project. There is no build step or framework — just HTML, CSS, and deployment via GitHub Actions.

## Commands

```bash
npm install              # Install dependencies (only js-yaml)
npm test                 # Run tests (bash script checking template files exist)
npx http-server docs -p 8000  # Local preview at http://localhost:8000
```

Generate a spec YAML from an external project directory:
```bash
node scripts/spec_from_project.js <project-path>
```

## Architecture

```
docs/
  index.html                      # Landing page with project cards
  assets/style.css                # Shared CSS (gitignored — used only by index.html)
  templates/
    sample-page.html              # Template with {{PLACEHOLDER}} tokens for new pages
    LLM_PROMPT.md                 # Prompt guide for LLM-based page generation
    index-template.html           # Template for the landing page
  pages/<project-id>/
    index.html                    # Project intro page (self-contained inline styles)
    <feature>.html                # Optional sub-feature pages

specs/                            # YAML spec files describing each project's metadata
scripts/
  spec_from_project.js            # Generates a spec YAML from a project's README
  test_page.sh                    # Test script (npm test entry point)
  deploy.sh / deploy_clean.sh     # Manual deploy helpers (commit docs/ and push)
  disabled/                       # Archived generator scripts (not used)
```

## Key Patterns

- **No automation/generators** — pages are created manually from templates or via LLM prompts, not generated by scripts. The `npm run generate` commands are intentionally disabled stubs.
- **Project pages use inline styles** for portability; only `docs/index.html` references `docs/assets/style.css`.
- **`docs/assets/` is gitignored** — `style.css` must exist locally for the landing page to render but is not tracked.
- **Google Fonts (Inter)** is used site-wide via CDN link.

## Adding a New Project Page

1. Create `docs/pages/<project-id>/index.html` from `docs/templates/sample-page.html` (or use LLM with `docs/templates/LLM_PROMPT.md`).
2. Add a project card to `docs/index.html` linking to `./pages/<project-id>/`.
3. Optionally create a spec in `specs/<project-id>.spec.yaml`.

## CI/CD

- **ci.yml** — Runs `npm test` on push when `docs/**`, `package.json`, or workflows change.
- **deploy-pages.yml** — Deploys `docs/` to `gh-pages` branch on any push touching `docs/` (uses `peaceiris/actions-gh-pages@v4`). Deploys from all branches, not just main.
- Deployment requires `GITHUB_TOKEN` (or `ACTIONS_DEPLOY_KEY` / `PAGES_PAT`) configured in repository secrets.

## Operational Notes (from Copilot instructions)

- Be conservative with CI/deploy workflow changes; test locally first.
- Do NOT commit `docs/` to `main` for publishing unless explicitly requested — CI deploys directly to `gh-pages`.
- When making workflow changes, explain required secrets and settings in PR descriptions.
